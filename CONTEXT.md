# 三角洲自动抢购工具 v2.0 — 领域术语表

## 核心概念

- **工作流（Workflow）**: 一组有序步骤的集合，定义了一个完整的自动化操作序列。存储在 `config/workflows.json` 中。
- **步骤（Step）**: 工作流中的单个操作单元。每个步骤有 `type` 字段标识类型，以及类型特定的参数。
- **步骤编辑器（Step Editor）**: 右侧区域，选中步骤后显示其所有可编辑字段。

## 步骤类型

- **tap**: 点击步骤。在设备指定坐标 `(x, y)` 执行单次点击。
- **long_press**: 长按步骤。在指定坐标执行长按。
- **swipe**: 滑动步骤。从 `(x1, y1)` 滑动到 `(x2, y2)`。
- **tap_point**: 精确点击步骤。带校准的坐标点击。

## 坐标系统

- **设备坐标（Device Coordinates）**: 设备物理分辨率下的像素坐标，如 2400×1080。步骤中存储的 `x`、`y` 即为设备坐标。
- **视图坐标（View Coordinates）**: 投屏画面在 UI 中的像素坐标。通过 `_view_to_device()` 按比例映射到设备坐标。

## 投屏与交互

- **投屏（Screen Mirror）**: 右侧区域，实时显示手机屏幕画面。基于 scrcpy 或 screencap 采集。
- **截图选择器（Screenshot Picker）**: 嵌入主窗口右侧的投屏组件，支持点击选点和标记。
- **坐标选择模式（Coordinate Pickup Mode）**: 一种特殊交互模式，用户在投屏上点击以获取设备坐标，用于填充步骤参数。当前仅对 tap 类型步骤可用。可选地同步在手机上执行 tap（通过勾选框控制）。

## 设备

- **设备分辨率（Device Resolution）**: 工作流级别记录的目标设备物理分辨率，如 `{"width": 2400, "height": 1080}`。
