## 需要实现的 7 个核心能力

### 能力 1：多任务系统
- 顶部有类似浏览器的标签栏，左侧是设置按钮，中间是任务标签，最右侧是"+"按钮
- 每个任务是一个独立的执行上下文：拥有自己的设备绑定、方案选择、步骤预览状态
- 点击"+"创建新任务（自动命名"任务N"），点击标签切换任务，关闭标签需二次确认
- 至少保留一个任务（不可全部删除）
- 首次启动自动创建"任务一"
- **任务状态持久化**：关闭程序再启动，所有任务和每个任务的设备/方案/步骤状态完整恢复
- 切换任务或关闭程序时提示保存

### 能力 2：设备绑定到任务
- 左侧边栏顶部显示设备信息：`(选择)/(重选) 手机型号 自定义名称 [重命名]`
- 选择设备后，该任务"认定"此设备，下次启动自动等待该设备连接
- 设备在线显示绿色状态，离线显示橙色等待状态
- 未选择设备或设备未连接时点击"启动"，弹出提示："无法启动，设备未找到或设备未被选择"

### 能力 3：方案管理
- 左侧边栏中部有方案切换下拉框，显示当前方案名称
- 下拉框右侧有"管理"按钮，点击弹出方案管理弹窗
- 弹窗底部有 5 个按钮：新建、删除、导出、导入、重命名
  - 新建：弹出配置窗口（名称 + 是否使用初始脚本预设），每个方案对应独立脚本
  - 删除：选中方案后删除，需二次确认，不可删除唯一方案
  - 导入：从系统文件管理器选择 JSON 文件，复制到引擎的方案目录
  - 导出：将当前选中方案导出为 JSON 文件到用户指定目录
  - 重命名：弹出文本输入框修改名称
- 方案变更后，所有引用处（下拉框、编辑面板）同步刷新

### 能力 4：步骤预览栏
- 左侧边栏下方有步骤预览区域，默认折叠模式（显示分组摘要）
- 双击可在折叠/展开之间切换
- 展开模式下，每个步骤单独一行显示：类型 + 备注/显示名称
- 点击某个步骤 → 右侧编辑器显示该步骤的属性表单

### 能力 5：步骤编辑器增强
- 右侧编辑器根据选中步骤类型动态渲染表单（已有能力，保留并扩展）
- 所有步骤类型新增"启用"开关和"显示名称"字段
- 新增"变量处理"步骤：三种互斥模式（布尔值 / 整数四则运算 / 字符串），切换模式时只显示对应字段
- 新增"ADB命令"步骤：命令文本输入 + 结果写入变量名
- 屏幕识别步骤增强：支持变量赋值（识别成功时设置变量值）
- OCR 步骤增强：支持结果写入变量 + 条件赋值（若识别结果包含某文本则设置变量）
- 修改字段后 400ms 防抖自动保存（避免高频写入磁盘）

### 能力 6：左下角控制按钮区
- 四排三列网格布局
- 第一排：20发模式开关（已开启/已关闭）、子弹数量（整数）、单发价格（小数）
- 第二排：从第一行启动、从选中步骤启动、停止/继续
- 第三排和第四排：预留按钮（禁用状态），后续扩展

### 能力 7：运行控制增强
- 启动前校验：任务存在 → 设备已选择且在线 → 方案已选择 → 步骤完整
- 校验失败弹出明确的中文提示
- 运行时变量存储：支持步骤间通过变量名传递数据

## 执行策略

1. **先读后写**：开始编码前，先阅读所有需要修改的文件的完整内容，理解现有信号连接和数据流
2. **自底向上**：先实现底层模型（TaskManager）→ 再实现 UI 组件（TabBar/DeviceBind/WorkflowSwitcher）→ 最后组装到 MainWindow
3. **每步可验证**：每完成一个能力后，尝试 `python main.py` 启动确认不崩溃
4. **保持信号连通**：新组件的信号必须正确连接到 MainWindow 的槽函数，确保数据流闭环
