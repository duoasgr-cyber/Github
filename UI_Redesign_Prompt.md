# 任务自动化控制台 — UI 重设计提示词

---

## 1. 角色设定

你是一位资深的 PyQt5 桌面端 UI/UX 重构工程师，精通 Qt 框架布局系统、信号槽机制和 QSS 样式体系。你擅长在不破坏现有数据格式和核心逻辑的前提下，对桌面应用进行信息架构重组和视觉升级。你正在帮助重构"三角洲自动抢购工具 v2.0"的用户界面。

## 2. 任务目标

### 最终要形成的新布局

将当前"左导航栏（设备+方案+步骤预览）+ 右主面板（堆叠面板+日志）"的二栏布局，升级为**四区现代工作台**布局：

`
+-----------------------------------------------------------+
|  顶部任务标签栏（标签页 + 设置 + 新建）                      |
+----------+------------------------------------------------+
|          |  顶部工具栏（方案切换 + 步骤操作按钮）             |
|  左侧栏  +----------------------------------+-------------+
|  信息区  |                                  |             |
|          |  主编辑区                        | 截屏选点区  |
| · 设备   |  (步骤列表 / 配置表单 /           | (截图预览 + |
| · 方案   |   设备管理 / 运行监控 /            |  坐标标注)  |
| · 步骤   |   测试面板)                       |             |
|   预览   |                                  |             |
|          +----------------------------------+-------------+
|          |  底部日志面板（可折叠）                           |
+----------+------------------------------------------------+
|  状态栏（设备/连接/OCR 状态）                               |
+-----------------------------------------------------------+
`

### 最终要支持的新交互模式

- **可折叠侧边栏**：左侧面板支持收起/展开（快捷键或按钮），收起后仅显示图标
- **可调节分隔**：各区域之间使用 QSplitter，用户可拖拽调整比例
- **步骤预览增强**：步骤列表支持分组折叠、拖拽排序、右键菜单（复制/删除/禁用）
- **截屏选点常驻**：截屏选点区域始终可见，不需要切换面板
- **运行状态浮动**：浮动窗口增加快捷操作（暂停/继续/停止）

### 最终要保留的旧能力

- 多任务标签管理（创建/切换/关闭任务）
- 设备绑定与状态监控
- 工作流（方案）编辑与管理
- 20+ 种步骤类型编辑
- 截屏选点与坐标校准
- OCR 识别与图像匹配
- 测试面板（单步/从某步/全流程）
- 日志过滤与清除
- 配置面板（购买参数、邮件、定时、识别等）
- 系统托盘与浮动窗口
- 任务状态持久化

## 3. 现状约束

### 必须遵守的约束

- 技术栈保持 PyQt5，不引入 QML 或 Web 渲染
- 保持 Python 3.10+ 兼容
- 所有现有信号槽连接关系必须保留或等价替换
- TaskStateManager 的 JSON 快照格式不可变（tasks.json 结构）
- ConfigManager 的配置键路径不可变
- 工作流 JSON 格式（steps 数组结构）不可变

### 不允许的改动

- 不得删除 core/ 目录下的任何引擎文件
- 不得修改 StepExecutor、WorkflowEngine、AdbCore 的内部逻辑
- 不得破坏已有的 JSON 配置文件格式
- 不得移除系统托盘功能
- 不得移除浮动窗口功能

### 兼容性要求

- 现有 config/*.json 文件必须可直接读取
- 现有 tp/ 模板图片目录结构不变
- 现有 you.txt 邮件计数文件不变
- 现有 QSS 样式可扩展但基础色板保持一致

## 4. 核心能力清单

### 4.1 可折叠侧边栏

- **用户场景**：编辑步骤时需要更多水平空间，希望收起侧边栏；需要查看设备/方案信息时展开
- **UI 行为**：
  - 侧边栏左侧有一个折叠按钮（◀ / ▶）
  - 收起时宽度从 260px 变为 48px，仅显示设备状态图标、方案名称缩写、步骤数量
  - 展开时恢复完整内容
  - 使用 QPropertyAnimation 实现平滑过渡（200ms）
  - 折叠状态持久化到 config/ui_state.json
- **验收标准**：折叠/展开动画流畅，状态记忆，不丢失任何信息

### 4.2 步骤列表增强

- **用户场景**：工作流有 20+ 步骤时，需要快速定位、批量操作
- **UI 行为**：
  - 步骤按类型自动分组显示（可选开关）
  - 支持拖拽排序（已有，保持）
  - 右键上下文菜单：复制步骤、删除步骤、禁用/启用、上移/下移
  - 选中步骤高亮 + 左侧彩色竖条标识步骤类型
  - 步骤类型用不同颜色区分（tap=蓝、wait=灰、ocr=绿、condition=橙）
- **验收标准**：20+ 步骤时操作流畅，分组/折叠不影响数据顺序

### 4.3 截屏选点常驻

- **用户场景**：编辑 tap/swipe 步骤时需要实时查看截图并选点，不想切换面板
- **UI 行为**：
  - 截屏区域始终显示在主编辑区右侧（与 WorkflowPanel 同级）
  - 编辑非坐标步骤时，截屏区显示"暂无坐标步骤"占位
  - 编辑坐标步骤时，自动刷新截图并高亮当前坐标点
  - 支持在截图上直接拖拽调整坐标
- **验收标准**：编辑步骤时无需切换面板即可选点

### 4.4 运行监控面板优化

- **用户场景**：运行工作流时需要实时查看进度、价格、邮件数
- **UI 行为**：
  - 运行状态用大号卡片展示（状态颜色 + 工作流名 + 当前步骤）
  - 价格和邮件数用仪表盘风格展示
  - 进度条显示步骤完成百分比
  - 浮动窗口增加"暂停/继续"和"停止"快捷按钮
- **验收标准**：运行时信息一目了然，浮动窗口可直接控制

### 4.5 日志面板折叠

- **用户场景**：不需要查看日志时希望释放垂直空间给主编辑区
- **UI 行为**：
  - 日志面板默认高度 180px（保持现有）
  - 添加折叠按钮，可收起为 36px 高度的标题栏
  - 折叠时有新日志到来，标题栏显示未读数量角标
  - 使用 QSplitter 实现可拖拽调整高度
- **验收标准**：折叠/展开不影响日志接收，角标正确

### 4.6 组件样式统一

- **用户场景**：不同面板的按钮、输入框、标签样式不统一
- **UI 行为**：
  - 统一按钮尺寸：主操作按钮高度 32px，次要按钮 28px
  - 统一字体：标题 Microsoft YaHei 11px Bold，正文 10px，代码 Consolas 10px
  - 统一间距：面板内边距 8px，组件间距 6px，组间距 12px
  - 为设备面板的 QTableWidget 添加 QSS 样式
  - 为所有 QHeaderView 添加样式
- **验收标准**：全应用视觉一致，无裸露的默认控件样式

## 6. 分阶段执行计划

### P1：侧边栏可折叠 + 布局骨架重构

## 可选增强方案

### 方案 A：紧凑工作台（推荐）

- 侧边栏可折叠 + 截屏常驻右侧 + 日志可折叠
- 优点：信息密度高，操作路径短
- 缺点：小屏幕下右侧截屏区可能较窄

### 方案 B：标签式全屏

- 去掉侧边栏，设备/方案信息移入顶部工具栏
- 优点：主编辑区最大化
- 缺点：设备状态不够直观

### 方案 C：三栏等分

- 左栏信息 + 中栏编辑 + 右栏截屏，三栏等宽
- 优点：布局对称美观
- 缺点：编辑区空间被压缩

**推荐方案 A**，理由：在不大幅改变用户习惯的前提下，通过可折叠和可调节分隔实现空间优化，改动量最小、风险最低。
