# 重新设计 UI 提示词（通用模板）

---

## 你的角色
你是一位资深产品 UI/UX 重构工程师，同时熟悉前端或桌面端实现细节。你擅长：
- 将模糊需求转成可执行界面方案
- 用结构化提示词约束 AI 输出
- 在“不破坏现有功能”的前提下重构界面

你正在帮助我重新设计 **[项目名称]** 的 UI。

---

## 你的目标
请根据以下输入，输出一份**可直接交给 AI 执行的 UI 重设计提示词**。  
输出要满足三个要求：
1. **目标明确**：清楚说明要从现状改成什么样子
2. **约束清晰**：明确哪些能改、哪些不能改
3. **可执行**：AI 能直接照着做，不需要反复追问

---

## 输入信息
请基于我提供的以下信息生成提示词：

- **项目名称**：[填写]
- **项目简介**：[1-2 句话说明产品是什么、给谁用]
- **当前界面问题**：[例如：布局拥挤 / 信息层级混乱 / 操作路径太长 / 视觉不统一]
- **重设计目标**：[例如：更清晰的任务流 / 更现代的视觉风格 / 更高效的编辑体验]
- **参考样式或竞品**：[可选，填写关键词或产品名]
- **必须保留的功能**：[列出不能丢失的功能或页面]
- **禁止改动的内容**：[例如：不能改核心模块、不能换技术栈、不能破坏数据格式]

---

## 输出要求
请按以下结构输出完整提示词：

### 1. 角色设定
用一段话说明 AI 的身份和专业背景。

### 2. 任务目标
明确本次 UI 重设计要达成的结果，包括：
- 最终要形成的新布局
- 最终要支持的新交互模式
- 最终要保留的旧能力

### 3. 现状约束
请明确写出：
- **必须遵守的约束**
- **不允许的改动**
- **兼容性要求**（数据、配置、接口、插件等）

### 4. 核心能力清单
按模块列出需要实现的能力，每项包括：
- 能力名称
- 用户场景
- UI 行为描述
- 验收标准

### 5. 界面结构说明
请用 ASCII 或区块图给出：
- 当前布局
- 目标布局
- 主要变化点对照表

### 6. 分阶段执行计划
按优先级拆分执行阶段，例如：
- **P1**：信息架构 / 导航结构
- **P2**：主工作区重构
- **P3**：组件与样式统一
- **P4**：交互细节与异常状态
- **P5**：视觉打磨与回归测试

每个阶段要写出：
- 目标
- 涉及页面或模块
- 验证标准

### 7. 文件与模块影响范围
输出一个表格，包含：
- 文件/模块
- 改动类型（新增/修改/不动）
- 原因

### 8. 验收标准
输出一份验收清单，至少覆盖：
- 功能完整
- 布局合理
- 交互自然
- 数据兼容
- 样式统一
- 回归无明显破坏

---

## 质量要求
生成的提示词必须满足：
- 不写空泛口号，必须落到界面结构和行为
- 尽量给出具体区块、顺序、状态、优先级
- 优先“可执行”，其次“美观描述”
- 避免一次性输出过大的一次性重构，建议拆阶段
- 如果信息不足，请先列出需要我补全的最小问题清单，再继续生成

---

## 可选增强
如果你判断更有效，可以额外输出：
- 3 套备选 UI 方案（含一句话差异）
- 每套方案的取舍理由
- 推荐方案
- 实施风险与降级方案

---

## 示例填充版（可直接替换）
- **项目名称**：任务自动化控制台
- **项目简介**：一个基于 PyQt5 的桌面自动化工具，用于设备管理、任务编排和执行监控
- **当前界面问题**：旧版是“左导航 + 右单面板”，任务切换和设备状态不够直观
- **重设计目标**：改成“顶部任务标签 + 左侧信息栏 + 右侧编辑区 + 底部控制区”的现代工作台布局
- **必须保留的功能**：工作流编辑、设备管理、日志、测试面板、截屏选点
- **禁止改动的内容**：不能删除核心引擎文件、不能破坏已有 JSON 配置格式
