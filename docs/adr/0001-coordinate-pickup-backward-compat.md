# ADR-0001: 坐标选择模式与投屏点击行为的向后兼容

## 状态

已接受

## 背景

为 tap 类型步骤新增"从投屏获取坐标"功能，引入显式的"坐标选择模式"（按钮触发、投屏变灰、提示文字、可选手机 tap）。

现有的 ScreenshotPicker 已有行为：任何时候点击投屏都会通过 `point_selected` 信号更新 StepEditor 的坐标字段。

## 决策

**保持现有行为不变**：不在坐标选择模式时，点击投屏仍然更新步骤的坐标字段。坐标选择模式是现有行为的"增强"（增加视觉反馈、提示、可选手机 tap），而非"替代"。

## 理由

1. **向后兼容**：不破坏用户已有的操作习惯
2. **渐进增强**：坐标选择模式提供更好的引导和额外功能（手机 tap），但基础功能始终可用
3. **低风险**：不需要修改 ScreenshotPicker 的核心信号流

## 替代方案

**严格模式**：只在坐标选择模式下才更新坐标。优点是交互意图更明确，缺点是改变了已有行为，且对熟练用户来说多了一步操作。

## 影响

- `StepEditor.update_coord_fields()` 的调用路径不变
- ScreenshotPicker 的 `point_clicked` → `point_selected` 信号链不变
- 新增的坐标选择模式通过 overlay 和状态管理实现，不影响底层信号流
