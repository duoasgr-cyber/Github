# 三角洲自动抢购工具 v2.0 — 设计文档

> 本文档为「三角洲自动抢购工具」从纯脚本升级为GUI应用的完整设计方案。

---

## 一、项目背景与目标

### 1.1 现状分析

当前项目（`D:\Desktop\PY`）是一个基于ADB+OCR+OpenCV的手机游戏自动化抢购脚本，核心功能包括：

- 通过ADB连接手机，执行点击、滑动、截屏等操作
- 通过OpenCV模板匹配识别游戏界面状态
- 通过EasyOCR识别游戏中的价格数字和中文文字
- 自动监控价格、在合适时购买道具
- 购买后执行「卡邮件」流程保留道具
- 自动售卖道具回收货币
- 悬浮窗实时显示价格和状态

### 1.2 现有痛点

| 编号 | 痛点 | 具体表现 |
|------|------|----------|
| P1 | **坐标全部硬编码** | 每个模块里 `shell input tap X Y` 的坐标写死在代码中，换设备/分辨率就要逐文件改代码 |
| P2 | **ADB命令函数重复拷贝** | `adb_command_basic()` 在 6 个文件中复制粘贴，修改一处需同步六处 |
| P3 | **配置散落** | 邮件数量用 `you.txt`，用户定价用 `用户定价.txt`，坐标用 `数据.json`，没有统一管理 |
| P4 | **无GUI配置入口** | 修改任何参数都要手动编辑文件或改代码，对非技术用户极不友好 |
| P5 | **硬编码绝对路径** | `C:\Users\Administrator\Desktop\PY\tp\xxx.jpg` 这类路径不灵活，换电脑就失效 |
| P6 | **操作流程不可编辑** | 购买、卡邮件、售卖等流程写死在代码逻辑中，增删步骤必须改代码 |
| P7 | **日志仅控制台输出** | print输出无法回溯，GUI中看不到运行信息 |
| P8 | **无错误恢复机制** | 流程中某步失败后无法从断点继续，只能从头开始 |

### 1.3 项目目标

1. **套壳**：为现有脚本套上PyQt5 GUI界面，所有参数可视化配置
2. **步骤可编辑**：将所有操作流程抽象为「步骤序列」，支持在GUI中增删改查、拖拽排序
3. **坐标可标注**：提供截屏选点功能，告别手动量坐标
4. **配置统一**：所有配置集中到JSON文件，GUI修改即时生效
5. **向后兼容**：核心业务逻辑保持不变，只是把硬编码值改为从配置读取
6. **可打包分发**：最终可打包为单个exe，无需用户安装Python环境

## 二、技术选型

| 项目 | 选择 | 理由 |
|------|------|------|
| GUI框架 | **PyQt5** | 原生美观、信号槽机制适合异步、支持QSS样式表美化、社区成熟 |
| 配置管理 | **JSON统一配置文件** | 结构化、可读性好、PyQt5可直接绑定、支持嵌套 |
| 打包分发 | **PyInstaller** | 一键打包成exe，支持资源文件打包，无需用户装Python |
| OCR引擎 | **EasyOCR**（保留） | 已有模型文件，识别效果已验证 |
| 图像匹配 | **OpenCV**（保留） | 模板匹配已稳定运行 |
| 截屏方式 | **ADB screencap**（保留） | 不依赖投屏软件，直接从设备获取像素级截图 |

### 2.1 为什么选PyQt5而不是其他方案

| 方案 | 优点 | 缺点 | 结论 |
|------|------|------|------|
| PyQt5 | 原生外观、信号槽异步、QSS美化、组件丰富 | 包体稍大（~30MB） | ✅ 推荐 |
| tkinter | 你已有经验、无需额外安装 | 美化困难、组件少、拖拽排序难实现 | ❌ |
| PySide6 | Qt官方维护、LGPL协议 | 与PyQt5 API略有差异、社区资源少 | ❌ |
| Web(Electron) | 界面自由度最高 | 包体巨大、Python-Node桥接复杂 | ❌ |
| DearPyGui | 轻量、GPU加速 | 生态小、自定义布局受限 | ❌ |

### 4.3 工作流步骤序列 — `config/workflows.json`

这是整个方案的核心数据模型，将所有操作流程从代码中提取为可编辑的步骤序列。

#### 4.3.1 步骤类型定义

| type | 必填参数 | 可选参数 | 说明 | 对应ADB命令 |
|------|----------|----------|------|-------------|
| `tap` | x, y | delay, desc | 点击屏幕 | `shell input tap X Y` |
| `long_press` | x, y | duration(默认3000ms), delay, desc | 长按 | `shell input swipe X Y X Y DURATION` |
| `swipe` | x1, y1, x2, y2 | duration(默认300ms), delay, desc | 滑动 | `shell input swipe X1 Y1 X2 Y2 DURATION` |
| `keyevent` | key | delay, desc | 按键事件 | `shell input keyevent KEY` |
| `wait` | duration | desc | 纯等待 | `time.sleep()` |
| `wifi` | enable | delay, desc | 开关WiFi | `shell svc wifi enable/disable` |
| `force_stop` | package | delay, desc | 强制停止应用 | `shell am force-stop PKG` |
| `launch` | package | delay, desc | 启动应用 | `shell monkey -p PKG -c ... 1` |
| `screenshot` | - | remote_path, delay, desc | 截屏到设备 | `shell screencap -p PATH` |
| `pull_file` | remote, local | delay, desc | 拉取文件 | `pull REMOTE LOCAL` |
| `delete_file` | remote | delay, desc | 删除设备文件 | `shell rm PATH` |
| `check_image` | template | threshold(默认0.85), delay, desc | 模板匹配判断 | 截屏+OpenCV matchTemplate |
| `ocr_region` | left, top, right, bottom | delay, desc | OCR识别区域 | 截屏裁剪+EasyOCR |
| `tap_point` | point_name | delay, desc | 引用坐标点名称点击 | 从coordinates.json读取坐标 |
| `call_workflow` | workflow | desc | 调用子工作流 | 递归执行另一个workflow |
| `condition` | check_type, check_param | true_workflow, false_workflow, desc | 条件分支 | 根据判断结果执行不同流程 |
| `loop` | count, workflow | desc | 循环执行 | 重复执行子工作流N次 |

### 5.1 ADB核心模块 — `core/adb_core.py`

消除现有6处重复的 `adb_command_basic`，提供统一的ADB操作接口。

```python
class ADBCore:
    """统一ADB命令执行器"""
    
    def __init__(self, device_serial=None):
        self.device_serial = device_serial
    
    def set_device(self, device_serial):
        self.device_serial = device_serial
    
    def run(self, command, timeout=30):
        """执行原始ADB命令"""
        if self.device_serial:
            cmd = f"adb -s {self.device_serial} {command}"
        else:
            cmd = f"adb {command}"
        return subprocess.run(cmd, shell=True, check=True, timeout=timeout)
    
    def tap(self, x, y):
        """点击屏幕坐标"""
        self.run(f"shell input tap {x} {y}")
    
    def swipe(self, x1, y1, x2, y2, duration=300):
        """滑动"""
        self.run(f"shell input swipe {x1} {y1} {x2} {y2} {duration}")
    
    def long_press(self, x, y, duration=3000):
        """长按（通过swipe实现）"""
        self.run(f"shell input swipe {x} {y} {x} {y} {duration}")
    
    def keyevent(self, key_code):
        """按键事件"""
        self.run(f"shell input keyevent {key_code}")
    
    def screenshot(self, remote_path="/sdcard/screen.png"):
        """截屏到设备"""
        self.run(f"shell screencap -p {remote_path}")
    
    def pull(self, remote_path, local_path):
        """从设备拉取文件"""
        self.run(f"pull {remote_path} {local_path}")
    
    def push(self, local_path, remote_path):
        """推送文件到设备"""
        self.run(f"push {local_path} {remote_path}")
    
    def shell_rm(self, remote_path):
        """删除设备上的文件"""
        self.run(f"shell rm {remote_path}")
    
    def force_stop(self, package):
        """强制停止应用"""
        self.run(f"shell am force-stop {package}")
    
    def launch_app(self, package):
        """启动应用"""
        self.run(f"shell monkey -p {package} -c android.intent.category.LAUNCHER 1")
    
    def wifi_enable(self, enable=True):
        """开关WiFi"""
        action = "enable" if enable else "disable"
        self.run(f"shell svc wifi {action}")
    
    def get_devices(self):
        """获取已连接设备列表"""
        result = subprocess.run("adb devices", capture_output=True, text=True, timeout=5)
        lines = result.stdout.strip().split('\n')
        devices = []
        for line in lines[1:]:
            parts = line.strip().split('\t')
            if len(parts) == 2 and parts[1] == 'device':
                devices.append(parts[0])
        return devices
    
    def screenshot_to_local(self, local_path="screenshot.png", region=None):
        """截屏并拉取到本地，可选裁剪区域"""
        remote = "/sdcard/_auto_screen.png"
        self.screenshot(remote)
        self.pull(remote, local_path)
        self.shell_rm(remote)
        
        if region:
            from PIL import Image
            with Image.open(local_path) as img:
                cropped = img.crop(region)
                cropped.save(local_path)
```

### 5.2 步骤执行引擎 — `core/step_executor.py`

读取JSON中定义的步骤序列并按序执行，支持暂停、停止、单步执行。

```python
class StepExecutor:
    """步骤执行引擎"""
    
    def __init__(self, adb, config_manager, callbacks=None):
        self.adb = adb                              # ADBCore实例
        self.config = config_manager                 # ConfigManager实例
        self.callbacks = callbacks                   # GUI回调
        self._running = False
        self._paused = False
        self._current_step_index = -1
    
    @property
    def current_step_index(self):
        return self._current_step_index
    
    def execute_workflow(self, workflow_name):
        """执行指定工作流"""
        workflow = self.config.get_workflow(workflow_name)
        if not workflow:
            raise ValueError(f"工作流不存在: {workflow_name}")
        
        self._running = True
        self._current_step_index = 0
        
        for i, step in enumerate(workflow["steps"]):
            if not self._running:
                break
            while self._paused:
                time.sleep(0.1)
            
            self._current_step_index = i
            self._notify_step_start(i, step)
            result = self._execute_step(step)
            self._notify_step_complete(i, step, result)
        
        self._running = False
        self._current_step_index = -1
    
    def execute_from_step(self, workflow_name, start_index):
        """从指定步骤开始执行"""
        workflow = self.config.get_workflow(workflow_name)
        self._running = True
        
        for i in range(start_index, len(workflow["steps"])):
            if not self._running:
                break
            while self._paused:
                time.sleep(0.1)
            
            self._current_step_index = i
            step = workflow["steps"][i]
            self._notify_step_start(i, step)
            result = self._execute_step(step)
            self._notify_step_complete(i, step, result)
        
        self._running = False
        self._current_step_index = -1
    
    def execute_single_step(self, step):
        """执行单个步骤（用于测试）"""
        return self._execute_step(step)
    
    def _execute_step(self, step):
        """执行单个步骤的核心逻辑"""
        step_type = step["type"]
        
        if step_type == "tap":
            self.adb.tap(step["x"], step["y"])
        
        elif step_type == "tap_point":
            point = self.config.get_point(step["point_name"])
            self.adb.tap(point["x"], point["y"])
        
        elif step_type == "long_press":
            self.adb.long_press(step["x"], step["y"], step.get("duration", 3000))
        
        elif step_type == "swipe":
            self.adb.swipe(
                step["x1"], step["y1"], 
                step["x2"], step["y2"], 
                step.get("duration", 300)
            )
        
        elif step_type == "keyevent":
            self.adb.keyevent(step["key"])
        
        elif step_type == "wait":
            pass  # 仅等待delay
        
        elif step_type == "wifi":
            self.adb.wifi_enable(step.get("enable", True))
        
        elif step_type == "force_stop":
            self.adb.force_stop(step.get("package", self.config.get("game_package")))
        
        elif step_type == "launch":
            self.adb.launch_app(step.get("package", self.config.get("game_package")))
        
        elif step_type == "screenshot":
            remote = step.get("remote_path", "/sdcard/_auto_step.png")
            self.adb.screenshot(remote)
        
        elif step_type == "pull_file":
            self.adb.pull(step["remote"], step["local"])
        
        elif step_type == "delete_file":
            self.adb.shell_rm(step["remote"])
        
        elif step_type == "check_image":
            # 截屏 → 拉取 → 模板匹配
            from core.screen_capture import ScreenCapture
            sc = ScreenCapture(self.adb)
            result = sc.match_template(
                step["template"], 
                step.get("threshold", self.config.get("similarity_threshold", 0.85))
            )
            return result
        
        elif step_type == "ocr_region":
            from core.ocr_engine import OCREngine
            ocr = OCREngine()
            text = ocr.recognize_region(
                step["left"], step["top"], step["right"], step["bottom"]
            )
            return text
        
        elif step_type == "call_workflow":
            return self.execute_workflow(step["workflow"])
        
        else:
            raise ValueError(f"未知步骤类型: {step_type}")
        
        # 执行delay等待
        delay = step.get("delay", 0)
        if delay and delay > 0:
            time.sleep(delay)
        
        return True
    
    def stop(self):
        self._running = False
    
    def pause(self):
        self._paused = True
    
    def resume(self):
        self._paused = False
    
    def _notify_step_start(self, index, step):
        if self.callbacks:
            self.callbacks.on_step_start(index, step)
    
    def _notify_step_complete(self, index, step, result):
        if self.callbacks:
            self.callbacks.on_step_complete(index, step, result)
```

### 5.3 工作流编排引擎 — `core/workflow_engine.py`

处理条件分支、循环等高级逻辑，对应现有 `mian.py` 中的主循环。

```python
class WorkflowEngine:
    """工作流编排引擎 — 处理主循环、条件分支等"""
    
    def __init__(self, adb, config, executor, callbacks=None):
        self.adb = adb
        self.config = config
        self.executor = executor
        self.callbacks = callbacks
        self._running = False
    
    def run_main_loop(self):
        """主循环：价格检查 → 购买 → 卡邮件"""
        self._running = True
        
        while self._running:
            # 1. 刷新价格
            self.executor.execute_workflow("refresh_price")
            time.sleep(1)
            
            # 2. 识别按钮文字
            from core.ocr_engine import OCREngine
            ocr = OCREngine()
            content_type, result_text = ocr.detect_button_text()
            
            # 3. 根据识别结果分支
            if content_type == "number":
                # 购买界面 — 进入价格检查
                self._notify_status("购买界面", "#4caf50")
                if not self._price_check_loop():
                    break
            
            elif content_type == "chinese" and "方" in result_text:
                # 确认购买界面
                self._notify_status("确认购买界面", "#4caf50")
                self.executor.execute_workflow("buy_confirm")
                time.sleep(2)
                if not self._card_mail_process():
                    break
            
            else:
                # 不在购买界面 — 恢复
                self._notify_status("恢复中...", "#ff6b6b")
                self._recover_game()
            
            time.sleep(0.5)
    
    def _price_check_loop(self):
        """价格检查循环"""
        while self._running:
            self.executor.execute_workflow("refresh_price")
            time.sleep(0.2)
            
            user_price = self.config.get("user_price")
            coefficient = self.config.get("price_coefficient")
            min_price = self.config.get("min_price")
            target = user_price * coefficient
            
            from core.ocr_engine import OCREngine
            ocr = OCREngine()
            game_price = ocr.get_price()
            
            self._notify_price(game_price)
            
            if game_price > 1000000000:
                break
            if game_price <= min_price:
                break
            if target >= game_price:
                self._notify_status("购买中...", "#4caf50")
                self.executor.execute_workflow("buy_confirm")
                time.sleep(2)
                if not self._card_mail_process():
                    return False
                break
        
        return True
    
    def _card_mail_process(self):
        """卡邮件完整流程"""
        self.executor.execute_workflow("after_buy")
        self.executor.execute_workflow("card_mail_begin")
        
        # 等待重启后检测界面
        while self._running:
            time.sleep(0.2)
            from core.screen_capture import ScreenCapture
            sc = ScreenCapture(self.adb)
            
            has_ru = sc.match_template("tp/kai_1.jpg")
            has_kai = sc.match_template("tp/kai_2.jpg")
            
            if has_ru or has_kai:
                break
        
        # 根据检测结果执行不同恢复流程
        if has_ru:
            self.executor.execute_workflow("card_mail_ru_run_1")
            self.executor.execute_workflow("refresh_price")
            self.executor.execute_workflow("card_mail_ru_run_2")
            
            # 检查邮件界面
            if sc.match_template("tp/yji.jpg"):
                if sc.match_template("tp/kt.jpg"):
                    self.executor.execute_workflow("card_mail_claim_kt")
                elif sc.match_template("tp/kf.jpg"):
                    pass  # 卡邮件失败
                else:
                    self.executor.execute_workflow("card_mail_claim_alt")
                
                self.executor.execute_workflow("card_mail_return")
                self.executor.execute_workflow("refresh_price")
        
        elif has_kai:
            self.executor.execute_workflow("card_mail_no_match")
            self.executor.execute_workflow("refresh_price")
        
        return True
    
    def _recover_game(self):
        """恢复游戏"""
        self.executor.execute_workflow("restart_game")
        
        while self._running:
            time.sleep(1)
            from core.screen_capture import ScreenCapture
            sc = ScreenCapture(self.adb)
            
            is_dl = sc.match_template("tp/dl.jpg")
            is_ru = sc.match_template("tp/kai_1.jpg")
            is_kai = sc.match_template("tp/kai_2.jpg")
            
            if is_dl or is_ru or is_kai:
                if is_ru:
                    self.executor.execute_workflow("recover_ru")
                elif is_kai:
                    self.executor.execute_workflow("recover_kai")
                elif is_dl:
                    self.executor.execute_workflow("recover_dl")
                break
    
    def stop(self):
        self._running = False
```

### 5.4 配置管理器 — `core/config_manager.py`

```python
class ConfigManager:
    """统一配置管理器"""
    
    def __init__(self, config_dir="config"):
        self.config_dir = config_dir
        self._config = None        # config.json
        self._coordinates = None   # coordinates.json
        self._workflows = None     # workflows.json
        self.load_all()
    
    def load_all(self):
        """加载所有配置文件"""
        self._config = self._load_json(os.path.join(self.config_dir, "config.json"))
        self._coordinates = self._load_json(os.path.join(self.config_dir, "coordinates.json"))
        self._workflows = self._load_json(os.path.join(self.config_dir, "workflows.json"))
    
    def _load_json(self, path):
        """加载JSON文件"""
        if not os.path.exists(path):
            return {}
        with open(path, 'r', encoding='utf-8') as f:
            return json.load(f)
    
    def _save_json(self, path, data):
        """保存JSON文件"""
        with open(path, 'w', encoding='utf-8') as f:
            json.dump(data, f, ensure_ascii=False, indent=2)
    
    # ---- config.json 操作 ----
    def get(self, key, default=None):
        return self._config.get(key, default)
    
    def set(self, key, value):
        self._config[key] = value
        self._save_json(os.path.join(self.config_dir, "config.json"), self._config)
    
    def get_all_config(self):
        return self._config.copy()
    
    def update_config(self, data):
        self._config.update(data)
        self._save_json(os.path.join(self.config_dir, "config.json"), self._config)
    
    # ---- coordinates.json 操作 ----
    def get_point(self, name):
        return self._coordinates.get("points", {}).get(name)
    
    def set_point(self, name, x, y, desc=""):
        if "points" not in self._coordinates:
            self._coordinates["points"] = {}
        self._coordinates["points"][name] = {"x": x, "y": y, "desc": desc}
        self._save_json(os.path.join(self.config_dir, "coordinates.json"), self._coordinates)
    
    def delete_point(self, name):
        self._coordinates.get("points", {}).pop(name, None)
        self._save_json(os.path.join(self.config_dir, "coordinates.json"), self._coordinates)
    
    def get_all_points(self):
        return self._coordinates.get("points", {}).copy()
    
    # ---- workflows.json 操作 ----
    def get_workflow(self, name):
        return self._workflows.get("workflows", {}).get(name)
    
    def get_workflow_names(self):
        return list(self._workflows.get("workflows", {}).keys())
    
    def save_workflow(self, name, workflow_data):
        if "workflows" not in self._workflows:
            self._workflows["workflows"] = {}
        self._workflows["workflows"][name] = workflow_data
        self._save_json(os.path.join(self.config_dir, "workflows.json"), self._workflows)
    
    def delete_workflow(self, name):
        self._workflows.get("workflows", {}).pop(name, None)
        self._save_json(os.path.join(self.config_dir, "workflows.json"), self._workflows)
    
    def get_all_workflows(self):
        return self._workflows.get("workflows", {}).copy()
```

2. **日志文件** (`app.log`) — 持久化存储，支持回溯排查

### 8.2 日志Handler

```python
class QTextEditLogHandler(logging.Handler):
    """将日志输出到PyQt5的QTextEdit"""
    
    def __init__(self, text_edit, max_lines=1000):
        super().__init__()
        self.text_edit = text_edit
        self.max_lines = max_lines
    
    def emit(self, record):
        msg = self.format(record)
        color = {
            'DEBUG': '#888888',
            'INFO': '#ffffff',
            'WARNING': '#ffaa00',
            'ERROR': '#ff6b6b',
            'CRITICAL': '#ff0000'
        }.get(record.levelname, '#ffffff')
        
        html = f'<span style="color:{color}">{msg}</span>'
        self.text_edit.append(html)
        
        # 限制行数
        if self.text_edit.document().lineCount() > self.max_lines:
            cursor = self.text_edit.textCursor()
            cursor.movePosition(cursor.Start)
            cursor.movePosition(cursor.Down, cursor.KeepAnchor)
            cursor.removeSelectedText()
```

## 十二、风险与注意事项

### 12.1 技术风险

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| EasyOCR模型加载慢 | 首次启动需10-30秒 | 启动时异步加载，显示加载进度条 |
| ADB连接不稳定 | 操作失败 | 每步执行前检查设备连接，失败自动重试 |
| PyQt5线程安全 | GUI卡顿或崩溃 | 所有ADB操作在子线程执行，通过信号槽更新UI |
| JSON配置损坏 | 程序无法启动 | 启动时校验JSON格式，损坏时自动恢复默认值 |
| 截屏选点坐标偏差 | 点击位置不准 | 严格按缩放比例换算，显示实际像素坐标供确认 |

### 12.2 兼容性注意

| 项目 | 注意点 |
|------|--------|
| Python版本 | 建议 3.9+，EasyOCR和PyQt5的兼容性最佳 |
| Windows版本 | Win10/11，ADB需在PATH中 |
| 手机分辨率 | 所有坐标基于特定分辨率，换设备需重新校准 |
| 游戏更新 | 游戏界面变化可能导致模板图片失效，需更新tp/目录 |

### 12.3 安全注意

| 项目 | 说明 |
|------|------|
| 配置文件 | 不存储任何账号密码，QQ登录由游戏客户端处理 |
| 日志文件 | 不记录敏感信息，仅记录操作流程 |
| 分发 | 打包时排除开发工具和调试信息 |
